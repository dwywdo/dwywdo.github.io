Gradle이란 무엇인가

  • Ref
    • https://www.youtube.com/watch?v=s-XZ5B15ZJ0&list=PL7mmuO705dG2pdxCYCCJeAgOeuQN1seZz&index=1
    • https://docs.gradle.org/current/userguide/userguide.html
  • 빌드를 수행하기 위한 빌드 시스템/도구/툴로, ant, maven과 같은 빌드 도구라고 볼 수 있다. 빌드를 위한 언어이다.
  • 개발을 수행할 때 직접적으로 와닿지는 않지만, 중요한 부분이다.
  • Java와 유사한 Groovy(Link)를 기반으로 한다.
    • Groovy: 자바 플랫폼을 기반으로 한 스크립트 언어.
    • 컴파일을 거쳐 실행할 수도 있고, 직접 스크립트로도 실행이 가능하다.
    • Java와 호환이 되기 때문에, Java로 작성된 소스코드를 Groovy에서 활용할 수도 있고, Groovy에서 자바 라이브러리를 사용할 수도 있다
  • maven이나 ant로 구성된 빌드시스템을 gradle로 손쉽게 바꿀 수 있다.
  • 자바 이외에 스칼라, 그루비, C/C++에 대해서도 빌드가 가능하다.
  • 오픈소스 기반으로, 새로운 기능이 추가 및 보완되고 있다.
  • 추가된 기능은 DSL(Domain Specific Language)을 통해서 제공된다.

DSL 활용 시 주의해야 할 부분

  • Ref: https://docs.gradle.org/current/dsl/index.html
  • 비공개(Internal): DSL / API로 제공 및 공개하지 않는 상태
  • 실험상태(Incubating): 기능은 추가되어 있으나, 개선 및 변경의 여지가 있는 상태
  • 공개상태(Public): 기능이 검증되어 활용이 가능한 상태
  • 폐지상태(Deprecated): Gradle의 버전이 업그레이드 되면서 없어질 예정이거나, 불필요하게 될 부분이 있는 상태
    • 이 상태에서도 사용할 수는 있으나, 언제 없어질 지 알 수 없다.
  • Ref: https://docs.gradle.org/current/dsl/org.gradle.api.Project.html
    • Core types의 Project 객체를 확인하면, 프로젝트 객체에서 제공하는 속성 / 관련 기능을 확인할 수 있다.
      • 이 기능들에 위의 상태들이 태그가 붙어 있는 것을 볼 수 있다.

Gradle의 빌드 수행 구성 요소

  • 초기화 스크립트 (init script): init.gradle - Gradle 객체와 연관됨
  • 설정 스크립트 (setting script) - settings.gradle - Settings 객체와 연관됨
    • 프로젝트 구조, 싱글인지 멀티프로젝트인지 등의 설정이 가능
  • 빌드 스크립트 (build script) - build.gradle - Project 객체와 연관됨
    • 빌드 수행과 관련되어 의존 관계, Task, Repository 등이 정의됨
    • Gradle의 기본 객체들을 통해 초기화 스크립트, 설정 스크립트의 내용 등을 참조할 수 있다.
  • 속성 파일: gradle.properties: 빌드 환경 속성에 대한 내용들이 존재. 보통 해당 프로젝트 하위에 위치한다.
  • 환경변수: 시스템에 등록한 Gradle 설치 정보
  • 명령어 옵션: 빌드 수행 시 명령어와 옵션 정보를 인수로 전달하는 것이 가능 
    gradle -Pname=gradle hello #name 변수에 gradle을 전달하면서 'hello' task 수행
  • 프로젝트 디렉터리(buildSrc): 빌드 수행 시 클래스 파일이나, 플러그인을 저장하여 참조하는 디렉터리

Gradle의 빌드 수행 단계

  1. 명령어 해석: 옵션 및 인수/디렉터리/속성 파일 등을 확인하고 클래스 인스턴스 생성
  • daemon, GUI, CLI 모드가 있으며, 명령어 옵션으로 구분하여 실행할 수 있다.
  • 두 가지 이상의 모드를 같이 사용하면, 내부적인 우선순위에 의해 선택된 모드로 실행됨 GUI > daemon
  1. Gradle 실행
  2. Script 초기화: Script 파일 읽음. 멀티 or 싱글 프로젝트 등을 판단하고, Project 객체를 생성
  3. Project Settings: 참조 중인 라이브러리 확인, Task 객체 및 Task Graph 생성
  • Task Graph는 Task 간의 의존관계를 나타냄
  1. Task Graph에 기반해서 Task를 실행
  2. 결과 출력

Gradle의 스크립트 파일 구조

  • Statement와 Script Block으로 구성된다.
  • Statement: 변수를 정의하는 부분이라고 생각할 수 있다.
def id = 'gradle'
  • Script Block: repository나, task 등 여러 프로젝트 빌드와 관련된 정보를 표현하는 데에 사용한다.
repositories {
    mavenCentral()
}

task idCheck<<{
    def id = 'check'
    println 'id: ' + id
}
  • repositories: 저장소 설정
  • dependencies: 의존 관계 설정
  • buildscript: 빌드 스크립트 클래스 패스 설정
  • initscript: 초기화 스크립트 설정
  • configurations: 환경 구성 설정
  • allprojects: 서브프로젝트 포함한 전체 프로젝트 설정
  • subproejcts: 서브프로젝트 설정
  • artifacts: 빌드 결과에 대한 설정

Gradle의 변수와 속성(Properties)

  • 지역 변수: 선언된 부분이 Scope인 변수. Gradle의 모든 스크립트에서 사용 가능
  • 시스템 속성: 시스템 관련 정보를 저장하는 변수. Gradle의 모든 스크립트에서 사용 가능
    • -D 옵션 또는 --system-prop 옵션 사용해서 지정 (gradle -D[속성명]=[속성값] OR gradle --system-prop [속성명]=[속성값])
    • gradle.properties에서 systemProp.[속성명]=[속성값]와 같이 설정 가능
    • 스크립트 내에서 접근할 때에는 System.properties['속성명']과 같이 사용 가능
  • 확장 속성: 도메인 객체의 확장 용도로 사용되는 변수. Gradle의 모든 스크립트에서 사용 가능
    • ext 키워드로 사용 가능하다.
    ext {
    extPro1 = 'pro1'
    extPro2 = 'pro2'
}
println ext.extPro1 // pro1
println ext.extPro2 // pro2
  • 프로젝트 속성: 프로젝트에서 사용되는 변수. 빌드 스크립트에서만 사용된다.
    • -P 옵션 또는 -project-prop 옵션 사용해서 설정 가능
  • 속성 파일과 명령어로 지정된 속성의 이름이 같으면 어떤 값을 사용하는가?
    • Ref: https://docs.gradle.org/current/userguide/build_environment.html#sec:gradle_configuration_properties
    • 우선순위가 존재한다.
      • 프로젝트 디렉터리의 gradle.properties
      • 홈 디렉터리의 gradle.properties
      • 환경 변수
      • -D 옵션
      • -P 옵션
